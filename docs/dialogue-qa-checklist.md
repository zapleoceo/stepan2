# Dialogue QA — чеклист регрессии диалогов

Живой список сценариев, на которых бот **уже ломался и был починен**. Прогоняй его
симуляцией после любой правки промпта / KB / reply-пайплайна. **Каждая новая
найденная-и-исправленная ошибка диалога → новая строка здесь** (сценарий + ожидаемое
поведение + тред-источник). Это регрессионный набор качества диалога.

## Где прогонять — ТОЛЬКО ClodeCouch (branch 8), НЕ Индонезия (branch 1)

Симуляции запускаются на **тест-филиале ClodeCouch (`branch_id=8`)**, а НЕ на боевой
Индонезии (`branch_id=1`). ClodeCouch связан с Индонезией через `kb_source_branch_id=1`
(см. [kb-sharing.md](kb-sharing.md)), поэтому `SimService` резолвит `effective_kb_branch`
и берёт **актуальную боевую базу знаний Индонезии** — персону MinStep, продукты, цены —
но пишет sim-треды в изолированный филиал, не засоряя реальный.

> ⛔ Никогда не гоняй sim на `branch_id=1` — это плодит фейковые треды в боевом филиале
> (портит отчёты/сегменты). Всегда `branch_id=8`.

Запуск (реальный reply-путь, `workflow='sim'`, не биллится):

```python
# в контейнере stepan2-api: python /tmp/sim.py
from app.modules.conversation.sim import SimService
from app.adapters.llm.broker import BrokerLLM
sim = SimService(session, BrokerLLM())
await sim.reset(8, 'qa-key')                 # 8 = ClodeCouch
r = await sim.say(8, 'qa-key', 'сообщение лида')
print(r['reply'], r['stage'], r['needs_manager'])
```

Многоходовые проверки: подавай реальные входящие лида по порядку. Для длинных живых
чатов — вытяни `SELECT text FROM message WHERE thread_id=X AND direction='in' ORDER BY
occurred_at` и прогони список через `sim.say(8, ...)`.

Детекторы/нуджи/приоритеты живут в `app/modules/conversation/situations.py` (один модуль,
одна цепочка `pick_nudge`); архитектура хода — [reply-pipeline.md](reply-pipeline.md).

**Prohibition-слой (guard):** глобальные баны живут в `policy_prohibitions` (always-injected);
`guard_verify` enforce-ит любую строку KB с `NEVER / does NOT / jangan / BUKAN`, даже если
похожие слова есть в KB. Единые источники политик: `policy_certificates` (BNSP), `policy_discounts`
(referral/pelajar/Vibe book-now), `policy_success_cases` (правила подачи кейсов). Карточки на них
ссылаются, не дублируют. Цены-канонизатор понимает «Rp X juta/ribu» + десятичную запятую.

## Как расширять

При каждой новой найденной ошибке диалога, после фикса и деплоя:

1. Добавь строку в таблицу выше: минимальная реплика лида, ожидаемое поведение,
   что НЕ должно быть, номер треда-источника.
2. Если фикс детерминированный (гейт в коде) — заведи юнит-тест
   (`tests/test_prompt_guardrails.py`, `test_discovery_cap_nudge.py`, `test_guard.py`).
3. Прогони затронутые строки чеклиста sim-ом на ClodeCouch перед мерджем.

Промпт-правила, на которые опирается чеклист, живут в
`app/modules/conversation/prompt.py`; методология — в
[sales-methodology.md](sales-methodology.md).
