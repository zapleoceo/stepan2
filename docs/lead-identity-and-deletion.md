# Личность лида и удаление канала

Как Stepan-2 создаёт/объединяет лидов и что происходит с данными при удалении канала.

## Модель личности лида

Один **лид** (`lead`) — это человек в рамках филиала. Один лид может иметь
несколько **тредов** (`channel_thread`) — по одному на каждый канал/чат, где он
писал (Instagram, WhatsApp, Meta Business).

Правило объединения — **по номеру телефона в пределах филиала**
([`IdentityService`](../app/modules/leads/identity.py)):

- Если у входящего есть телефон и в филиале уже есть лид с таким `phone_e164` →
  это **тот же лид** (даже если пишет из другого канала). Тред привязывается к нему.
- Иначе, если тред уже существует → берём его лида.
- Иначе создаём нового лида.

Недостающие поля (имя, `ig_user_id`, `ig_username`, аватар) **дозаполняются**, но не
перезатираются (кроме аватара — CDN-URL протухает). Изоляция по `branch_id` — в
`BranchScoped`-репозиториях; лиды разных филиалов никогда не сливаются.

### Извлечение телефона по стране филиала

`phone_e164` строит [`extract_phone`](../app/modules/leads/phone.py) из сырого текста.
Форма мобильного номера **зависит от страны** — она берётся из настройки филиала
`phone_country_code` (`62` Индонезия по умолчанию, `60` Малайзия, `63` Филиппины):

- ищется только форма ЭТОЙ страны (`8[1-9]…` / `1[0-9]…` / `9…`), поэтому цена
  (`Rp 1.200.000`) не даёт ложных совпадений;
- локальный `0…`-номер получает **свой** префикс страны (`+60`, `+63`), а не хардкод `+62`;
- номер чужой формы или неизвестный код страны → `None` (никогда не мис-префиксится),
  так что кросс-филиальное объединение по телефону остаётся безопасным.

**Следствие:** канал НЕ владеет лидом. Лид с телефоном может держать треды сразу в
IG и WA. Поэтому удаление одного канала не может слепо удалять его лидов.

## Каскад при удалении канала

[`ChannelService.purge`](../app/modules/channels/service.py) удаляет канал и все
висящие на нём данные в **FK-безопасном порядке**, в одной транзакции запроса
(при сбое — полный откат, канал остаётся цел). Всё branch-scoped и параметризовано.

Порядок:

1. `media_asset` сообщений канала → `message` канала;
2. `outbox`, `manager_alert`, `stage_event` тредов канала → `channel_thread` канала;
3. **лиды-сироты** — у кого не осталось НИ ОДНОГО треда ни в одном канале филиала —
   вместе с их `manager_alert` / `stage_event`;
4. `channel_session` → `channel`.

**Ключевой инвариант:** лид, объединённый по телефону и имеющий тред в другом
канале филиала, **выживает** — удаляется только его тред/сообщения на удаляемом
канале. Лид, живший лишь на удаляемом канале, удаляется целиком. Это исключает
осиротевшие треды с висящим `channel_id` и конфликты при повторном подключении
аккаунта. Покрытие: [`tests/test_channel_purge.py`](../tests/test_channel_purge.py).

Роут удаления — [`_routes_channels.channel_delete`](../app/api/_routes_channels.py)
(tenant-guard `_channel_branch` → `ChannelService.purge`).
