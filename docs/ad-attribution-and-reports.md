# Атрибуция рекламы и авто-привязка продукта

Как лид, пришедший по рекламе, получает продукт в чате ещё до того, как Степан начнёт
квалификацию, и как этим управляет оператор.

## Что приходит на входе

Приватный IG-адаптер вытаскивает контекст рекламы из инбокса и кладёт его на тред
(`channel_thread`), см. `app/adapters/channels/transports.py`:

- `ad_id` — ID объявления в Meta Ads Manager (`ad_context_data.ad_id`);
- `ad_media_id` — IG media ID креатива (`professional_metadata.ad_ig_media_id`);
- `ad_preview_url` — CDN-превью креатива (часто пустой);
- `lead_source` — `story` | `ad_clicktomsg` | пусто.

Запись этих полей — `app/modules/leads/ingest.py:_store` (только на первом входящем).

## Маппинг реклама → продукт

Meta Marketing API мы не читаем, поэтому имя/кампанию объявления программно не знаем.
Источник истины — **оператор-заданная таблица** `ad_product_map` (`ad_id → product_slug`,
per-branch), сервис `app/modules/ads/mapping.py`.

- `product_for_ad(ad_id)` — какой продукт рекламирует это объявление;
- `upsert / clear` — правки оператора;
- `suggest_from_history()` — **подсказка** для UI: самый частый `product_slug` среди
  прошлых тредов этой рекламы. Никогда не пишется в таблицу автоматически — это
  самоусиливающийся сигнал (отражает прошлые догадки Степана), только префилл.

## Провенанс продукта (`channel_thread.product_source`)

Продукт на треде может прийти из трёх источников; колонка `product_source` хранит,
откуда именно, и это гейтит переопределение:

| Источник | Значение | Кто ставит |
|---|---|---|
| Реклама | `ad` | ingest: `product_hint` из пейлоада или `ad_product_map` |
| Модель | `model` | `ReplyService._apply_decision` при уверенной квалификации |
| Менеджер | `manager` | ручная смена в UI чата (`_routes_chat.py`) |

Правило переопределения (`app/modules/conversation/reply.py:_apply_decision`): модель
может переквалифицировать продукт, унаследованный от рекламы (`ad`) или от своего же
прошлого хода (`model`), но **никогда не перебивает ручной выбор менеджера** (`manager`).
Так реклама даёт стартовый дефолт, Степан уточняет по диалогу, а решение человека —
финально.
