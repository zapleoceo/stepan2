# Открытие чата: что грузится и почему быстро

`GET /ui/chat/{thread_id}` (`_build_chat_panel` в
[`_routes_chat.py`](../app/api/_routes_chat.py)) рендерит панель чата пятью запросами к
Postgres (шапка лида, сообщения, pending-очередь, события/лог, активные продукты филиала) —
без обращения к брокеру и без N+1.

## Перевод "потребностей" (needs) — кеш + фоновая загрузка

Шапка чата показывает захваченный VPC-профиль лида (`jobs/pains/gains`, см.
[sales-methodology.md](sales-methodology.md)), переведённый на язык интерфейса менеджера.
Раньше перевод шёл **синхронно внутри рендера страницы** — первое открытие треда с
непереведённой фразой ждало живой вызов брокера прямо в GET-запросе.

Теперь рендер панели зовёт [`cached_needs`](../app/modules/conversation/needs_translate.py) —
чистую функцию без I/O, которая читает только уже закэшированное в `lead.needs_tr` и
возвращает `(profile, pending)`. Если `pending=True` (есть непереведённая фраза), блок
needs рендерится сразу с фолбэком на оригинальный текст и несёт
`hx-get="/ui/chat/{tid}/needs" hx-trigger="load"` — браузер сам догружает реальный перевод
после того, как страница уже отрисована. Обработчик `chat_needs_lazy` (там же,
`_routes_chat.py`) — единственное место, которое теперь зовёт брокер
(`translated_needs`) и пишет `lead.needs_tr`.

`_needs_for` (тоже в `_routes_chat.py`) — старый синхронный путь, оставлен только для этого
lazy-эндпоинта; больше не вызывается из рендера страницы.

## Медиа-превью сообщений — один JOIN вместо 2×N подзапросов

`fetch_messages`/`fetch_messages_since` ([`_query.py`](../app/api/_query.py)) раньше
резолвили превью медиа двумя коррелированными подзапросами **на каждую строку сообщения**
(`media_id`, `media_kind`) — на длинном треде это Postgres выполнял отдельно для каждого
сообщения. Заменено на один `LEFT JOIN` с `ROW_NUMBER() OVER (PARTITION BY message_id ...)`,
считающий "первый media_asset с данными" один раз для всего треда — тот же результат,
но без квадратичного роста стоимости с длиной треда. Работает и на Postgres, и на SQLite
(оконные функции — 3.25+), так что тесты не завязаны на диалект.
