# Free reply mode — свободный режим продаж

Переключаемый режим генерации ответов: вместо скриптового контракта (13 ходов,
discovery-лестница, turn-notes, pitch/answer-гейты, критик) — короткая цель + ПОЛНАЯ
фактовая поверхность, и сильная модель сама решает, как продавать. Жёстким остаётся
только money-gate.

## Включение

Настройка филиала `reply_mode` (`app/modules/settings/schema.py`): `scripted` (дефолт,
классическое поведение, ничего не меняется) | `free`. Переключение мгновенное
(30 с TTL кэша настроек) — это и канарейка, и откат одной кнопкой.

## Цель (зашита в контракт `free_mode.py`)

1. Довести лида до **явного согласия** записаться на курс.
2. Взять телефон/WhatsApp — менеджер звонит в рабочие часы 09:00–18:00 WIB (не дозвонился →
   WhatsApp); вне этих часов бот обещает контакт «с 9 утра следующего рабочего дня».
3. Лид готов платить сейчас → бот сам даёт варианты оплаты из KB, не паркует горячего лида.

## Структура промпта — кэшируемый префикс

`build_messages_free` (`app/modules/conversation/free_mode.py`):

- `messages[0]` (system) — **байт-стабильный префикс**: полная KB
  (`KnowledgeService.full_knowledge_context`: persona + все facts-доки целиком без
  гейтирования секций + весь objection_playbook + ВСЕ карточки продуктов, сортировка по
  slug) + free-контракт + схема. Одинаков между ходами и между лидами одного
  филиала/языка — это якорь prompt-кэша брокера. Стабилен в пределах локального дня
  (единственный датозависимый вход — `annotate_dates`).
- `messages[1]` (system) — всё переменное: now-block, manager rules/note, entry-hint, имя,
  LEAD DOSSIER, first-turn note.
- Дальше диалог.

**Инвариант:** никаких условных вставок в `messages[0]` — любая ломает кэш и множит счёт
Sonnet. Закреплено тестом `test_prompt_prefix_is_byte_stable_across_leads_and_turns`.
Потолок префикса — `free_context_char_budget` (90k, `app/config.py`).

## Роутинг

`pick_capability` (структурный, без изменений) выбирает tier; в free-режиме
`SMART → chat:sales` (Sonnet-first цепочка брокера), `FAST → chat:fast` как раньше.
Фолбэк: транспортная ошибка/кап на `chat:sales` → повтор на `chat:smart`
(`_generate_free`) — деградация к сегодняшнему качеству, не к молчанию.
`chat:sales` добавлен в `_SLOW_CAPS` брокер-адаптера (длинный JSON-профиль).

## Что отключено в free (и живо в scripted)

- 9 turn-notes + bare-ack either-or шаблон (детерминированные костыли под слабые модели)
- pitch-gate, answer-gate, критик
- skeleton-opener для typed-входов (полный free-пайплайн пишет открытие сам)
- enum-валидация `move`: модель даёт свой snake_case-ярлык (телеметрия), парсер
  `parse_turn_decision(free_moves=True)` не приводит к списку `MOVES`

## Что остаётся жёстким в обоих режимах

- **Money-gate целиком** (`money_issues`, fail-closed): цены/ссылки/доход/выдуманные услуги
  только из KB; один rewrite на `chat:sales`, затем hold-line + эскалация.
- Чисто шаблонные openers (AD_SILENT/STORY/JUNK) — анти-бан, ноль LLM.
- Досье: merge + discovery-backstop — на нём фолоапы, роутинг, CRM, аналитика.
- Вся доставка: бабблы, капы, тихие часы, стадии, hand-off, CRM-push.

Фолоапы/реактивация пока остаются на скриптовом пути в обоих режимах.

## Метрики выката (sim A/B, branch 8)

Baseline scripted vs free по: (1) доля диалогов с явным согласием, (2) доля согласившихся,
отдавших телефон, (3) доля дошедших до цены + ходов до согласия. Выкат free только при
free ≥ scripted по (1)–(2). Sim — **только ClodeCouch branch 8**, никогда branch 1.
